<h1>Introduction</h1>
Today I will be looking into how the options selected on Rufus affect the data structure of a USB as well as how I can verify the correct changes were made.
In this article, I will be using FOX621_SAMPLEFW_1 with the following options selected when formatting it through Rufus:
- Boot selection: **Non Bootable**
- Partition scheme: **MBR**
- File system: **FAT32**
- Cluster size: **8192 bytes**

<h1>Boot selection</h1>

<h2>MBR + Non Bootable</h2>

As I have selected **‘Non Bootable’** and using MBR as the partition scheme, there won’t be any content in the first sector or what some may call the first physical sector, sector 0 , etc. This occurs as the first 446 bytes of the first physical sector actually points to the system’s starting partition location on the disk which won’t be needed here due to me selecting **‘Non Bootable’**.

Firstly, FTK Imager shows that the first 446 bytes contain no content as there is no operating system installed. Secondly, Autopsy shows that there is no operating system under Data Artifacts. Thirdly, using file manager software such as Windows File Explorer or in this case File Pilot shows that the EFI folder and other directories do not exist.

<h2>MBR + Bootable</h2>

If we select it to be **'Bootable'** and flash an .iso on it such as Fedora, Debian, etc., the first 446 bytes of the first sector will have content in it which points to the system's starting partition. A folder such as **'/EFI'** will also be created as it stores the boot files to boot up the operating system that we just flashed onto it.

<h1>Partition scheme</h1>

A partition scheme is the data structure that determines how a storage device (which is the USB in this case) is structured so operating systems, software, etc. know where to find the data. There are generally two: **GPT** and **MBR**, but I will be covering mainly MBR in this article. 

<h2>MBR</h2>

The MBR is located at the first physical sector of a storage device and to verify it, we can always look for the MBR end signature which is **55 AA**.

{% include figure image_path="/assets/images/BLOG_1v2_USB/Figure7_new.png" alt="MBR non-bootable" caption="Screenshot of FOX621_SAMPLEFW_1 MBR first 446 bytes via FTK imager" %}

<h2>GPT</h2>
GPT is vastly different compared to MBR in terms of its structure (heavily simplified version):
- Protective MBR
- Primary GPT Header
- Primary Partition Table
- Usable Partition Entry Array
- Backup Partition Entry Array
- Backup GPT header

By inspecting the first partition entry in the MBR, if the value of byte offset 4 is **0xEE**, this indicates that the storage device is using GPT. 

{% include figure image_path="/assets/images/BLOG_1v2_USB/Figure8_new.png" alt="MBR non-bootable" caption="Screenshot of FOX621_SAMPLEFW_1 MBR first 446 bytes via FTK imager" %}

<h1>File system</h1>

We can easily tell the file system based on the content in the file system.

<h2>FAT + MBR</h2>

As this is using the MBR partition scheme, we will be examining the partition table to verify the file storage system used here. We can also use a forensic tool such as Autopsy to inspect the files inside the partition that contains the file storage system to identify the type of file storage system it is using.
