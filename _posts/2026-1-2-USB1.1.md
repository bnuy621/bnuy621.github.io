This is a short continuation to my previous blog post , we missed out the option ‘Target System’ and how it affects the structure of the storage device / USB in this case.

<h1>Target system</h1>
Well, this option does not **directly** affect the data structure. Instead, it indirecetly affects the data structure by limiting the choices we can make for the options that do directly affect the data structure of a storage device (USB in )

<h2>Bootable</h2>

<table>

    <td>Fedora-KDE-Desktop-Live-43-1.6x86_64.iso’</td>

    <td>MBR</td>
    <td>FAT32</td>
    <td>4096B, 8192B, 16KB, 32KB, 64KB</td>
  </tr>
  <tr>
    <td>Fedora-KDE-Desktop-Live-43-1.6x86_64.iso’</td>
    <td>UEFI (non csm)</td>
    <td>GPT</td>
    <td>FAT32</td>
    <td>4096B, 8192B, 16KB, 32KB, 64KB</td>
  </tr>
</table>
As you can see, the target system for a bootable USB / storage device does affect the partition scheme in which the **partition scheme ‘MBR’** is exclusive to the choice **‘BIOS or UEFI’** while the partition scheme **‘GPT’** seems to be exclusive to **‘UEFI (non CSM)’**  However, both still only allow **FAT32** as the file system.
